## What Is This

**Investment News is a locally-run "investment-sector news dashboard" — it uses your own LLM to distill the latest from 100+ authoritative sources across 12 investment sectors into a daily set of key points (in Chinese) plus translations, shown in a browser dashboard.**

It's not yet another endless news aggregator — the core is **AI reading and distilling for you**: 3–5 "today's key points" at the top of each sector, aggregated and deduplicated across sources, so you grasp the whole picture at a glance and click ↗ for the original only when interested. Fetching runs **on your machine** (pure Python stdlib), the AI uses **your own** Claude subscription ($0) or any API key, and **data never leaves your machine — no account, no hosting.**

Why you'd want it:

- You want to track a bunch of investment sectors, but **there's too much news to read, and most of it is in English**.
- Portals and aggregators give you a pile of fragments; what you want is "what actually happened in this sector today" — which is exactly what the AI key-points layer does.

> 📊 **It's a news/trends tool (leading signal), not market data, and certainly not financial advice.**

## ✨ Features

| Feature | Description |
|---|---|
| **AI daily key points** | 3–5 key points at the top of each sector, aggregated & deduped across sources, highlighting companies/numbers/trends — generated by **your own LLM** |
| **Bilingual** | English titles auto-translated; each card shows a large Chinese title + the original in smaller text |
| **12 investment sectors** | AI · Semiconductors · Robotics · EV/Auto · Energy · Biotech/Health · Space · Cybersecurity · Tech · Consumer Electronics · Finance/Macro · Science |
| **Sourced** | Every key point carries a ↗ link to the original article it's based on |
| **One-click refresh** | The ⟳ button (top-left): the backend runs fetch + AI, spins while working, then auto-updates — no command line needed |
| **Subscription / API** | `claude-cli` (local Claude subscription, $0) or any OpenAI-compatible API (DeepSeek/OpenAI/SiliconFlow/OpenRouter…), switched by one config |
| **Local & zero-key** | Fetching is pure Python stdlib; data stays on your machine — no database, no hosting, no RSSHub |
| **Compliance filter** | Auto-filters gambling / prediction-market / crypto / adult content; politics and finance are kept |

## 🚀 Quick Start

**Requirements**: Python 3.7+ (stdlib only — **no pip install needed**) + one LLM (pick one below).

```bash
git clone https://github.com/simonlin1212/investment-news.git
cd investment-news

# 1) Configure the LLM (see "Configure" below) — defaults to your local Claude subscription, $0
# 2) Start the dashboard
python3 server.py            # default port 8793, keep it running
# 3) Open the dashboard
open http://localhost:8793   # Windows: start, Linux: xdg-open
# 4) Click ⟳ (top-left) to refresh — AI fetches + summarizes + translates, then auto-updates
```

## ⚙️ How It Works

```
sources.json  (108 sources / 12 sectors)
       │
       ▼  scripts/fetch.py    fetch + redline filter + recent-N-days + Beijing time
  data.js  (raw items)
       │
       ▼  scripts/digest.py   your LLM → per-sector "key points" + Chinese translation + source links
  data.js  (with AI key points)
       │
       ▼  index.html          browser dashboard (single file, no build, no deps)
  server.py: local server + /api/refresh for the refresh button
```

All **pure Python stdlib + one LLM** — no database, no RSSHub, no hosting. In `claude-cli` mode, `digest` spawns the local `claude -p` (subscription auth, all tools disabled, text-only) — **local-only, $0**.

## 🤖 Configure your LLM (subscription / API)

Edit `llm.config.json`:

| provider | Description | Cost |
|---|---|---|
| **`claude-cli` (default)** | Uses your logged-in **Claude Code subscription** (`claude login` once), local-only | **$0** |
| **`api`** | Any **OpenAI-compatible API** (DeepSeek / OpenAI / SiliconFlow / OpenRouter…), runs anywhere | usage-based |

```jsonc
{ "provider": "claude-cli" }                       // subscription — nothing else to fill
// or
{ "provider": "api",
  "api": { "base_url": "https://api.deepseek.com", "api_key": "sk-...", "model": "deepseek-chat" } }
```

## ➕ Add a source = add one line

Add a line to the `sources` array in `sources.json`, no code changes:

```jsonc
{ "name": "Some Media", "hint": "ai", "type": "rss", "url": "https://example.com/feed" }
```

`hint` = sector key (ai / semi / robot / auto / energy / bio / space / security / tech / consumer / macro / science).
`fetch.recent_days` controls the recency window (default 7); `redline_keywords` is the compliance filter.
